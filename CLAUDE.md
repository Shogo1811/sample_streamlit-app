# Project: sample_streamlit-app

物流システムのStreamlitアプリケーション（Snowflake連携）

## AI-DLC ワークフロー
このプロジェクトは `aws-aidlc-rule-details/` のルールに従って開発する。

## カスタムコマンド

- `/review` — マルチ専門家レビュー（自動修正サイクル付き。不合格時は自動修正→再レビューを最大3回繰り返し、全専門家90点以上を目指す）
- `/expert-review` — マルチ専門家レビュー（reviewと同一内容）
- `/generate-spec` — お客様提出用の仕様書を実装コードから生成（旧版は日付フォルダにアーカイブ）
- `/generate-manual` — お客様向けマニュアルを実装コードから生成（旧版は日付フォルダにアーカイブ）

## Pre-Push チェック
`git push` 前にリンター（ruff）、フォーマッター（ruff format）、テスト（pytest）が自動実行される。
- Git hook: `.githooks/pre-push`
- Claude Code hook: `.claude/settings.json` の PreToolUse で `git push` をインターセプト
- チェックスクリプト: `scripts/pre-push-checks.sh`

## 承認ポリシー（信頼度ベース判断）
- AIの判断に自信がある場合は、ユーザーへの確認なしに自動続行する
- 判断に迷う場合、または重大な設計決定（アーキテクチャ変更、破壊的変更等）のみユーザーに確認を求める
- INCEPTIONフェーズの各ステージ間遷移では、矛盾や曖昧さがない限り承認を求めない
- CONSTRUCTIONのコード生成計画（Part 1）は概要のみ提示し、異議がなければ続行する

## ドキュメント生成ポリシー（必須/任意の区別）
ユーザーが確認すべきドキュメント（**必須確認**）:
- 要件定義の主要決定事項（箇条書き5行以内で提示）
- コード生成計画の概要（ファイル一覧 + 実装方針）
- レビュー結果のスコアサマリー

AI内部用ドキュメント（**確認不要** — 自動生成・自動管理）:
- `aidlc-docs/audit.md`（監査ログ）
- `aidlc-docs/aidlc-state.md`（状態管理）
- 質問ファイル・clarificationファイル（回答後は参照不要）
- 詳細設計ドキュメント（AI実装の根拠用）

## コード生成後ウォークスルー
コード生成（Part 2）完了後に、以下の形式でシステム全体の概要を提示する:
1. **生成ファイル一覧** — 各ファイルの役割を1行で日本語説明
2. **依存関係** — モジュール間の呼び出し関係を簡潔に図示
3. **処理フロー** — ユーザー操作から画面表示・DB書込までの主要フローを説明
※ コード生成中の途中確認は不要。全生成完了後にまとめて説明する。

## レビューポリシー（根拠明示）
- `/review` の各専門家の指摘には、**根拠**（公式ドキュメントURL、ベストプラクティス名、法令条文等）を付記すること
- 根拠を示せない指摘は「推奨事項」として明確に区別し、合否判定の対象外とする
- ユーザーが専門外の分野（例: Legal, QA）の指摘を評価しやすいよう、なぜその指摘が重要かの1行説明を添える

## ディレクトリ規約
- `aidlc-docs/` — AI-DLCドキュメント（設計書、状態管理、レビュー結果）
- `docs/specifications/` — お客様提出用仕様書（最新版のみ、旧版は archive/ 配下）
- `docs/manuals/` — お客様向けマニュアル（最新版のみ、旧版は archive/ 配下）
- `aws-aidlc-rule-details/` — AI-DLCルール定義（変更しない）
- `scripts/` — ユーティリティスクリプト
