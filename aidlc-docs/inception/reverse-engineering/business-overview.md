# ビジネス概要

## システム名
ラーメンチェーン店 食材物流管理システム

## ビジネス目的
ラーメンチェーン店舗の食材在庫・発注・配送を一元管理し、店長とドライバーの業務効率を向上させる。

## 主要機能
1. **在庫確認（FR-01〜05）** — 店舗別在庫のリアルタイム可視化、閾値アラート
2. **発注承認（FR-06〜10）** — 自動生成された発注提案の確認・承認・却下ワークフロー
3. **配送管理（FR-11〜14）** — 配送ステータス追跡、ドライバー向け完了報告

## ユーザーロール
| ロール | 主な操作 |
|---|---|
| 店長（MANAGER） | ダッシュボード閲覧、発注承認/却下、配送状況監視 |
| ドライバー（DRIVER） | 配送一覧確認、配送完了報告 |

## データフロー
```
食材マスタ → 在庫管理 → 発注提案（自動生成）→ 店長承認 → 発注予定
                                                    ↓
                                              配送割当 → ドライバー完了報告
```

## 監査・コンプライアンス
- 全書込操作を AUDIT_LOG に記録（操作者、日時、変更前後値）
- プライバシー同意の取得・履歴管理（CONSENT_RECORDS）
- Row Access Policy (RLS) によるデータアクセス制御
