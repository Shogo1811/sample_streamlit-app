# User Personas

## Persona 1: 田中店長（店長ロール）

| 項目 | 詳細 |
|------|------|
| **役割** | ラーメンチェーン店の店長 |
| **年齢層** | 30〜50代 |
| **管理範囲** | 自分の担当する1店舗のみ |
| **技術レベル** | 基本的なPC操作は可能（Excel、メール等）。業務システムの利用経験は限定的 |
| **主要デバイス** | PC / タブレット（店舗バックヤードで利用） |
| **利用環境** | 店舗バックヤードの事務スペース。Wi-Fi接続 |
| **利用頻度** | 毎日1〜2回（朝の在庫確認、発注承認時） |

### 目標
- 食材の在庫切れを防ぎ、営業に支障を出さない
- 発注業務の手間を減らし、調理や接客に時間を使いたい
- 配送状況を把握し、食材の到着予定を確認したい

### 課題
- 在庫確認を手作業で行っており、漏れが発生しやすい
- 発注のタイミングや数量を経験と勘に頼っている
- 配送業者への電話確認に時間がかかる

### システムへの期待
- 画面を見ればすぐに在庫状況がわかること
- 発注提案が自動で出てきて、確認・承認するだけで済むこと
- 操作が直感的で、マニュアルなしでも使えること

## Persona 2: 佐藤ドライバー（ドライバーロール）

| 項目 | 詳細 |
|------|------|
| **役割** | 食品配送業者のドライバー |
| **年齢層** | 20〜60代（幅広い） |
| **担当範囲** | 割り当てられた配送先への食材配送 |
| **技術レベル** | スマートフォンの基本操作は可能（LINE、地図アプリ等） |
| **主要デバイス** | スマートフォン（配送先で利用） |
| **利用環境** | 屋外・配送先（店舗裏口等）。モバイル回線（4G/LTE） |
| **利用頻度** | 配送のたびに利用（1日に複数回） |

### 目標
- 当日の配送先とスケジュールを素早く確認したい
- 配送完了を簡単に報告し、次の配送先に向かいたい
- 操作に時間をかけたくない

### 課題
- 配送先の確認に紙のリストや電話連絡を使っており非効率
- 配送完了の報告が手作業（電話、メモ）で漏れることがある
- 運転中や荷降ろし中のため、複雑な操作ができない

### システムへの期待
- スマホでさっと配送先リストが見られること
- 配送完了はボタン1つで報告できること（入力作業なし）
- 画面がシンプルで、見やすく・押しやすいこと

### SiS環境でのUX注記
- Snowsight経由でのスマートフォンアクセスとなり、UXは最適ではない（ベストエフォート）
- 配送完了操作がボタン1つのため、SiS制約下でも実用上問題なし
- 画面はst.columnsの単一カラムレイアウト・大型ボタンで設計
