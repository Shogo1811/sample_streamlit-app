# User Story Generation Plan

## Overview
ラーメンチェーン店食材自動発注システムのユーザーストーリーを、INVEST基準に準拠して作成する。

## Methodology
- **Breakdown Approach**: エピック単位（Q7=A）
- **Story Format**: 「〜として、〜したい。なぜなら〜だから。」形式
- **Acceptance Criteria**: Given-When-Then形式

## Personas
- **店長**: 基本PC操作可能（Q1=B）、PC/タブレット利用、1店舗管理（Q3=A）
- **ドライバー**: スマートフォン利用（Q2=A）、配送完了はボタン操作のみ（Q6=A）、SiS経由でアクセス（UXベストエフォート）

## Q&A Summary
| Q | 回答 | 設計反映 |
|---|------|----------|
| Q1 | B: 基本PC操作可能 | 標準UI複雑さでOK |
| Q2 | A: スマートフォン | ドライバー画面はシンプル＋大型ボタン。SiS制約によりUXベストエフォート |
| Q3 | A: 1店舗のみ | 店舗切替UI不要 |
| Q4 | A: 保存のみ手動連絡 | CSVエクスポート不要 |
| Q5 | A: システム固定閾値 | 閾値設定画面不要 |
| Q6 | A: ボタンのみ | ドライバー画面は配送リスト＋完了ボタンのみ |
| Q7 | A: エピック単位 | 大きな機能グループ単位 |

## Execution Checklist

### Part 1: Personas
- [x] 店長ペルソナの定義（役割、目標、課題、技術レベル: B、デバイス: PC/タブレット、利用環境: 店舗バックヤード）
- [x] ドライバーペルソナの定義（役割、目標、課題、技術レベル: スマホ基本操作、デバイス: スマートフォン、利用環境: 屋外/配送先、ネットワーク: モバイル回線）

### Part 2: Epic - 可視化ダッシュボード（FR-01〜FR-05）
- [x] US-1.1: 店長として、自店舗の在庫状況・在庫推移・配送状況をダッシュボードで確認したい
- [x] US-1.2: 店長として、期間・食材カテゴリでデータをフィルタリングしたい
- [x] US-1.3: 店長として、在庫が閾値を下回っている食材のアラートを確認したい

### Part 3: Epic - 発注提案・承認（FR-06〜FR-10）
- [x] US-2.1: 店長として、発注提案リストを確認し、承認/却下/数量修正を行いたい
- [x] US-2.2: 店長として、スケジュールベースの定期発注提案を確認したい
- [x] US-2.3: 店長として、承認済みの発注予定リストを確認したい

### Part 4: Epic - 配送管理（FR-11〜FR-14）
- [x] US-3.1: ドライバーとして、担当配送先リストとスケジュールをスマホで確認したい
- [x] US-3.2: ドライバーとして、配送完了ボタンを押して報告したい（ボタン操作のみ）
- [x] US-3.3: 店長として、自店舗への配送状況を確認したい

### Part 5: Epic - 認証・プライバシー（FR-17〜FR-19）
- [x] US-4.1: 店長/ドライバーとして、ログイン後に自分のデータのみにアクセスできる
- [x] US-4.2: 共通：初回利用時にプライバシーポリシーに同意する
- [x] セキュリティ：他店舗/他ドライバーのデータへのアクセスが拒否される（ネガティブケース — US-4.1に含む）

### Part 5.5: Epic - 横断的UI/エラーハンドリング
