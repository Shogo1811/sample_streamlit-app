# お客様向けマニュアル生成

あなたはテクニカルライターです。現在の実装コードを正（Single Source of Truth）として、お客様が利用するマニュアルを生成してください。

## 前提条件
- マニュアルはコードの実装を正として作成する
- エンドユーザー（技術者ではない場合もある）が理解できる平易な言葉で記述する
- 画面キャプチャの代わりにASCIIダイアグラムや手順の詳細説明を使用する

## 実行手順

### Step 1: 既存マニュアルの保全

1. `docs/manuals/` ディレクトリに既存のマニュアルがあるか確認する
2. **既存マニュアルがある場合**:
   - `docs/manuals/archive/{YYYYMMDD}/` ディレクトリを作成
   - 既存のマニュアルをすべてアーカイブディレクトリにコピー
   - コピー完了後、元のマニュアルを削除
   - `docs/manuals/archive/{YYYYMMDD}/README.md` に以下を記録:
     ```markdown
     # Archived Manuals
     **Archive Date**: {YYYY-MM-DD HH:mm:ss}
     **Reason**: New manual generated from current implementation
     **Files**:
     - {アーカイブしたファイル一覧}
     ```
3. **既存マニュアルがない場合**: Step 2 に進む

### Step 2: コードベース分析

1. ワークスペース全体のソースコードを読み取る
2. 以下を分析・整理する:
   - アプリケーションの全機能
   - 画面構成と画面遷移
   - ユーザーが実行できる操作
   - 入力項目とバリデーションルール
   - 出力・表示内容
   - エラーメッセージと対処法

### Step 3: マニュアル生成

`docs/manuals/user-manual.md` に以下の構成でマニュアルを作成する:

```markdown
# {プロジェクト名} ユーザーマニュアル

**バージョン**: {バージョン番号}
**作成日**: {YYYY-MM-DD}
**対象読者**: {対象ユーザーの説明}

---

## 1. はじめに
### 1.1 このマニュアルについて
### 1.2 システムの概要
### 1.3 動作環境・推奨ブラウザ

## 2. ログイン・初期設定
### 2.1 アクセス方法
### 2.2 ログイン手順（該当する場合）
### 2.3 初期画面の説明

## 3. 基本操作
### 3.1 画面構成の説明
### 3.2 共通操作（ナビゲーション、フィルタ、ソート等）

## 4. 機能別操作ガイド
（機能ごとに:）
### 4.X {機能名}
#### 概要
#### 操作手順（ステップバイステップ）
#### 入力項目の説明
#### 出力・結果の見方
#### 注意事項

## 5. トラブルシューティング
### 5.1 よくあるエラーと対処法
### 5.2 FAQ

## 6. 用語集

## 7. お問い合わせ先
```

### Step 4: 専門家レビュー

マニュアルの生成が完了したら、以下の専門家サブエージェントを **並列で** 起動しレビューする:

1. **Frontend Expert** — UI説明の正確性、画面遷移の網羅性
2. **QA Expert** — 手順の正確性、再現可能性、エッジケースの記載
3. **Legal Expert** — 免責事項、個人情報取り扱い説明の適切性
4. **Technical Writer Expert** — 文章の平易さ、構成の論理性、用語の一貫性、読みやすさ

各専門家は100点満点でスコアリングし、90点以上を合格とする。
不合格の場合は指摘事項を修正し、再レビューを行う。

### Step 5: 完了メッセージ

```markdown
> **📖 マニュアル生成完了**
> マニュアル: `docs/manuals/user-manual.md`
> {アーカイブした場合: 旧版アーカイブ: `docs/manuals/archive/{YYYYMMDD}/`}
>
> **レビュー結果**: {PASS or FAIL}
> {結果の要約}
>
> 人間による最終確認をお願いします。
```

## 重要なルール
- 実装コードを正とする（実際のコードから操作手順を導出する）
- 既存マニュアルは必ず日付フォルダにアーカイブしてから新規作成する
- AIが古いマニュアルを再度読み込んで混乱しないよう、アーカイブは別領域に保全する
- 技術用語は用語集で定義し、本文中では平易な表現を使う
- 操作手順は必ずステップバイステップで記述する
- 専門家レビューで90点未満の項目があれば修正してから完了とする
